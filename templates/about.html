System design

From prompt to protected database action.

PromptDB accepts a user prompt, generates a strict JSON command, validates the command, and executes the matching MongoDB operation.

Prompt -> LLM JSON -> Validation -> MongoDB -> UI
01

Input validation

The backend checks empty messages, length limits, allowed collections, allowed fields, and supported MongoDB operators.

02

AI command generation

OpenRouter is instructed to return only JSON, keeping the database layer separate from natural language.

03

Database execution

MongoDB operations run only after the parsed command passes the backend guardrails.

04

Operational resilience

MongoDB connections are lazy, timed out, and surfaced as clear UI errors when external services are unavailable.
